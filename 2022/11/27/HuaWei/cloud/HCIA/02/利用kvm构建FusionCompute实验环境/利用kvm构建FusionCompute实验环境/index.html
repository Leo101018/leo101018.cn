利用kvm构建FusionCompute实验环境

发表于 2022-11-27 分类于 HuaWei ， cloud ， HCIA 阅读次数：
本文字数： 4.4k 阅读时长 ≈ 4 分钟
利用物理机上kvm中创建vm，然后在vm中部署FusionCompute实验环境，并且可以在CNA中正确创建vm，即使用了嵌套虚拟化方式，很多网友说FusionCompute不支持嵌套虚拟化，实际通过实验证明，可以支持。

kvm的vm中部署FusionCompute过程

架构决定了该处理器的新旧程度，比如Intel的第二代酷睿i系列架构为Sandy Bridge，第三代架构为Ivy Bridge，第四代架构为Haswell和Crystallwell。

一般预设的硬体是较旧的 pc-i440fx 晶片， 但是新的应该使用 q35 比较好！

物理机：10.15.2.71，系统为rocky Linux 8.7版本

安装kvm环境：

yum install qemu-kvm libvirt virt-install virt-manager virt-viewer dejavu-lgc-sans-fonts -y

Transaction Summary

=========================================================================================================================

rocky-Linux 8.7 默认就开启了嵌套虚拟化状态：

cat /sys/module/kvm_intel/parameters/nested

[root@10-15-2-71 ~]# cat /sys/module/kvm_intel/parameters/nested

1

[root@10-15-2-71 ~]#

CNA

部署vm，安装CNA：

[root@10-15-2-71 CNA01]# qemu-img create -f qcow2 CNA01-01.qcow2 200G

将主板芯片组改成Q35，这个是比较新的芯片组。

CPU选择直通host的CPU。

系统盘我们选择virtIO总线，不作为本地存储硬盘。

网卡这里选择e1000，不要选择virtIO型号，否则VRM无法添加CNA主机。

增加一块硬盘，作为CNA的本地存储。

使用SATA总线类型。不要使用SCSI或者virtIO总线类型，否则VRM中扫描本地硬盘存储的时候，扫描不到本地硬盘。

然后编辑vm的xml配置文件，增加bios支持。

[root@10-15-2-71 CNA01]# virsh list –all

Id Name State

-——————————

3 win10 running

5 centos7.9_01 running

- CNA01 shut off

[root@10-15-2-71 CNA01]# virsh edit CNA01

点击开始安装。

原因是光驱使用的是IDE模式的，而Q35主板芯片组不支持IDE模式。

这里改成SATA总线类型。

sda是SATA总线类型的硬盘。

vda是virtIO总线类型的硬盘

bios信息。

系统信息。

主板信息。

处理器芯片。

由于cpu采用的是直通模式，所以，这里看到的是物理宿主机host上的cpu信息。

VRM

部署vm，安装VRM

[root@10-15-2-71 VRM]# qemu-img create -f qcow2 VRM-01.qcow2 200G

同理，Q35芯片组不支持IDE，因此，需要改成SATA模式。

使得在xml界面中可以直接编辑保存。

点击开始安装，过程同CNA模式。

宿主物理机host配置端口转发，以便远程可以访问：

iptables -t nat -A PREROUTING -p tcp –dport 443 -j DNAT –to-destination 192.168.122.119:8443

iptables -t nat -A POSTROUTING -p tcp -d 192.168.122.119 –dport 8443 -j SNAT –to-source 192.168.122.1

增加SNAT使得物理机上的kvm虚机可以通过nat上网。

iptables -t nat -A POSTROUTING -s 192.168.122.0/24 -j MASQUERADE

admin默认口令：IaaS@PORTAL-CLOUD8!

可以看到，添加CNA的时候，就是在对应CNA上创建ovs网桥。

注意，这里要能够看到cpu，bios，主板等这些信息，否则说明当前环境不支持嵌套虚拟化，那么就没有办法创建vm了。

扫描本地存储设备。

之前增加的SATA总线硬盘。

vlan0，就是无标签数据包，即数据包不打vlan tag，因此，相当于是桥接了。

上传ISO镜像到本地存储。

上传文件过程是首先存放在.uploads目录中，临时文件，待完全上传之后，会放入在.userfiles文件夹中。

在这里也可以进行上传。

正常安装系统即可。

网络使用的是nat方式。

因为使用的是vlan0方式。

tap即是vm的接口。

192.168.122.0是物理机的nat网段，因此，有dhcp，并且能够进行snat，物理机上的iptables配置了。

因此，vm中的ip为122段dhcp分配，网关为122.1，即物理机上的nat网关。

因为我们使用的是vlan0，因此，vm和CNA之间就是桥接了，即没有vlan tag。

现在CNA中建立的vm可以上外网。

安装bzip2工具包，目的是安装vm-tools时解压bzip压缩包。

目前没有安装Tools工具，因此，对于vm的一些监控信息和查看的信息，都无法准确查看，因此，要安装Tools工具。

reboot重启。

vm安装了tools之后，相关vm的监控信息就可以看到了。

可以在物理机host上做端口转发，使得可以远程连接到CNA中的vm。

iptables -t nat -A PREROUTING -p tcp –dport 22044 -j DNAT –to-destination 192.168.122.122:22

iptables -t nat -A POSTROUTING -p tcp -d 192.168.122.122 –dport 22 -j SNAT –to-source 192.168.122.1

此时就连接到了CNA中的vm了。

vm中下载安装一些软件，可以看到网络有流量了。
