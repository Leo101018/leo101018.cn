centos7.9.2009下VirtualBox网络配置

VirtualBox安装

安装图形gnome界面

yum groupinstall “GNOME Desktop” “Graphical Administration Tools” -y

设置默认运行级别为图形：systemctl set-default graphical

查看当前运行级别：systemctl get-default

无需重启立即开启图形界面：systemctl isolate graphical.target

安装必要的头文件以及编译工具等

yum install kernel-devel-$(uname -r) kernel-headers-$(uname -r) gcc -y

下载软件包[root@localhost Downloads]# ls

VirtualBox-7.0-7.0.2_154219_el7-1.x86_64.rpm

[root@localhost Downloads]#

nat自定义网络

该nat网络默认vm中的数据包可以通过nat出到宿主机做原地址转换，到的公网。但是无法通过宿主机直接登录到vm的。

对于nat，宿主机上没有相应的虚拟网络设备能够与vm进行通信。

vm可以ping通宿主机IP。通过nat做网关，进行了原地址转换。

反之不行。

而对于自定义nat网络

两台vm选择同一个自定义的nat网络。

即是这样的模型。

那么为何有VirtualBox提供的默认的nat和自定义的nat两种方式：

原因是各个vm可以都选择网络模式为默认的nat，那么各个vm都有自己的nat，彼此之间即使看上去都是同一个网段但是也互不相通，并且，各个vm都是独立可以上外网的。

而自定义的nat，可以自定义多个不同网段的nat网络，那么不同vm的网络模式可以选择不同的nat网络，使用不同的nat网络内的dhcp自动获取ip，这样，各个vm之间网络隔离了，不互相通。同时各个vm也可以选择相同的同一个nat网络，这样各个vm之间是相通的，并且都可以上外网。

同时两种方式，都可以设置端口转发，实现宿主机连到vm。

host-only网络

针对host-only网络创建完毕之后，会在宿主机上创建对应的虚拟网络设备，用于宿主机和vm之间互通。

另外，注意，host-only直接创建时，指定的网络只能是默认的网络，21位掩码，不能指定其它网段，查询官方文档：

128是指上面创建host-only，可以最多创建128个，那么宿主机上最多只能有128个host-only的虚拟网络设备。

编辑该文件，取消host-only对网段的限制。

之后，host-only这里就可以自定义网段了。并开启dhcp。

如果不进行宿主机上networks.conf设置，那么host-only这里自定义网段，开启dhcp，点击应用会报错，如下：

Error: Host Only Network- Failed to save host network interface parameter - E_ACCESSDENIED

此时宿主机的该虚拟网络设备就不是no-carrier了。

此时宿主机和vm之间可以互通。

仅主机网络对于预先配置的虚拟设备特别有用，其中多个虚拟机一起运行并设计用于协作。例如，一个虚拟机可能包含一个 web 服务器，另一个虚拟机包含一个数据库，因为它们打算彼此通信，所以该设备可以指示 VirtualBox 为这两个虚拟机建立一个仅限于主机的网络。第二个网络是桥接的，然后将 Web 服务器连接到外部世界以提供数据，但外部世界无法连接到数据库。
